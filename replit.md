# Jipe Life — MLM Platform

A full-stack binary MLM platform for Jipe Global Service Limited with E-Pin registration, binary tree matching engine, CTO royalty pools, lifetime rewards, member dashboard, and admin control panel.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/jipe-life run dev` — run the frontend (port 18116, path: `/`)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/scripts run seed-admin` — seed admin + demo users
- Required env: `DATABASE_URL` — Postgres connection string, `SESSION_SECRET` — session secret

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5, in-memory session auth (Bearer tokens)
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec → React Query hooks + Zod schemas)
- Frontend: React + Vite, Tailwind CSS v4, shadcn/ui, wouter routing
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — OpenAPI spec (source of truth for all endpoints)
- `lib/api-client-react/src/generated/api.ts` — generated React Query hooks
- `lib/api-zod/src/generated/api.ts` — generated Zod request/response schemas
- `lib/db/src/schema/` — PostgreSQL schema (users, epins, wallets, binary, cto, rewards)
- `artifacts/api-server/src/routes/` — all Express route handlers
- `artifacts/jipe-life/src/pages/` — all frontend pages (member + admin)

## Architecture decisions

- **In-memory sessions**: A Map stores session tokens → userId/role. Simple, works for single-server deployment. Replace with Redis for multi-server.
- **Binary engine as cron**: The binary matching engine runs on demand via admin panel (`POST /api/admin/cron/run-binary`). Should be triggered daily via a cron job in production.
- **CTO distribution**: Monthly CTO royalty distribution runs via admin panel (`POST /api/admin/cron/run-cto`). Admin must first record turnover via `POST /api/admin/turnover`.
- **E-Pin gated registration**: Users cannot register without a valid, unused E-Pin. Pins are generated by admin and assigned/transferred.
- **15% withdrawal deduction**: 10% admin fee + 5% TDS on all withdrawal payouts. Amounts are held pending → released on approval.

## Product

- **Member Portal**: Dashboard with balance/BV/pairs overview, binary engine stats + history, team tree visualization (3 levels), CTO pool with 100% recovery tracking, 9-tier lifetime reward milestones, E-Pin manager (view/transfer/copy), wallet with withdrawal requests and deduction preview, profile with bank/UPI details.
- **Admin Portal**: Dashboard KPIs, user management (search/block/unblock), bulk E-Pin generation + assignment, withdrawal approval/rejection workflow, binary engine manual cron trigger, CTO pool turnover recording + monthly distribution trigger.

## Packages & Business Logic

- **Smart**: ₹2,100 / 1 BV — CTO pool: 5% of monthly turnover
- **Silver**: ₹5,200 / 2 BV — CTO pool: 10% of monthly turnover
- **Gold**: ₹10,100 / 4 BV — CTO pool: 15% of monthly turnover
- **Binary**: 1:1 matching ratio; ₹1,000/pair; cap 4 pairs/day = ₹4,000; pairs 5-9 flushed (power leg carry-forward); 10 pairs = jackpot ₹7,000; 11+ = ₹200/pair unlimited
- **Rewards**: 9 milestones from 25 pairs (T-Shirt / ₹500) to 50,000 pairs (Tata Sierra / ₹11.5 Lakh) — claimable as physical reward or cash

## Gotchas

- Always run `pnpm run typecheck:libs` before running api-server if schema changed
- The DB schema uses Drizzle `pgEnum` — changing enum values requires a migration
- Binary matching and CTO distribution are idempotent per day/month — running twice on the same day/month will skip or error
- `REWARD_MILESTONES` is exported from `@workspace/db` (in `lib/db/src/schema/rewards.ts`)
